# myCPU 进度汇报 PPT 大纲（建议 12 页）

## 第 1 页：封面

- 题目：myCPU：从 0 实现可运行程序的 RISC-V 指令集模拟器
- 副标题：五级流水 + I/D Cache + MMU + 中断外设
- 信息：课程、姓名、日期

讲述重点：一句话说明目标是“做一台可独立运行的虚拟计算机”。

## 第 2 页：项目目标与范围

- 目标 1：完成可运行 myCPU 模拟器（本阶段主目标）
- 目标 2：为后续 miniOS 预留硬件抽象接口（可选）
- 对标：Bochs 思路，RISC-V 架构教学实现

讲述重点：强调“硬件设计者视角”，不是只做解释器。

## 第 3 页：总体架构

- CPU 核心：Decoder + Pipeline + CSR + RegisterFile
- 内存系统：Memory + MMU + I/D Cache
- 外设系统：UART + Timer + TOHOST
- 工具链：ELF Loader + Trace + Web Dashboard + 自动测试

建议展示：一张模块框图（可手画）。

## 第 4 页：CPU 微结构实现

- 五级流水：IF/ID/EX/MEM/WB
- 关键机制：前递、load-use 冒险暂停、分支冲刷
- 支持步进和断点调试模式

讲述重点：说明流水线不是“顺序模拟”，而是带冒险控制的周期推进。

## 第 5 页：指令集实现进展

- 已实现指令识别 49 条（不含占位标签）
- 覆盖类别：
  - 算术逻辑：ADD/SUB/SLT/位运算/移位等
  - 访存：LB/LH/LW/LBU/LHU + SB/SH/SW
  - 控制流：BEQ/BNE/BLT/BGE/JAL/JALR
  - 系统：ECALL、MRET、SRET、CSR、FENCE、SFENCE.VMA

讲述重点：满足课程 30-40 条要求并有扩展。

## 第 6 页：内存、Cache 与 MMU

- 物理内存 + 越界检测 + MMIO 映射
- I/D Cache 分离，支持配置容量、组相联、写策略、miss penalty
- Sv32 两级页表 + TLB + SFENCE.VMA

建议展示：Cache 统计字段和命中率输出样例。

## 第 7 页：异常、中断与外设

- trap 流程：ECALL/页故障/中断进入 + MRET/SRET 返回
- 中断源：Timer、UART
- 外设：UART 串口输出、Timer MMIO、TOHOST 退出码

讲述重点：说明已经具备 OS 所需的最小硬件接口雏形。

## 第 8 页：正确性验证结果

- CTest：19/19 通过
- rv32ui：p1/p10/no-cache 均 42/42 通过
- 数据口径说明：p1 CSV 重复行已清洗，三组口径统一为 42 项唯一测试

讲述重点：结果可信，且已完成一次数据治理闭环。

## 第 9 页：性能结果与收益

- matmul：cache 相比 no-cache 约 11.34x 加速
- quicksort：约 12.71x 加速
- 观察：I-cache 命中率对整体性能影响显著

建议展示：已有报告中的 3 张图（speedup、hitrate、benchmark 对数图）。

## 第 10 页：工程化与可视化能力

- 一键脚本：批量测试 + CSV + 报告生成
- Trace JSON + SSE + Web 看板
- 可用于课堂演示流水线状态与访存事件

讲述重点：项目不仅“能跑”，还“可观测、可复现”。

## 第 11 页：差距与风险

- 已知差距：
  - 尚无独立中断控制器模块（当前为简化直连）
  - MMU 仍是基础版，特权细节可继续增强
  - 暂未交付 miniOS 镜像
- 环境说明：
  - WSL1 可运行当前工程，建议脚本化启动与 --quiet 跑性能，减少环境噪声
- 风险应对：
  - 持续执行 CSV 唯一性校验，避免统计口径回退

讲述重点：主动说明边界，体现项目管理能力。
